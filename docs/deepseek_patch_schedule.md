# 修正指示：GitHub Actions 定時実行スケジュールの前倒し

現在すでに実装済みのシステムにおいて、一部の要件が変更となりました。
既存のコードベースを維持したまま、以下の指示に従ってピンポイントで修正を行ってください。

## 【修正の背景と目的】
ユーザーが「毎日朝8時」にWikiで最新ニュースをチェックできるようにするため、システム処理のタイムラグを考慮し、バッチの開始時刻を「JST午前9時」から「**JST午前7時50分**」へ前倒しします。

## 【修正対象と指示内容】
対象ファイル： **`.github/workflows/update_wiki.yml`**

1. `schedule` トリガー内の `cron` 指定を以下のように変更してください。
   - **変更前**: `0 0 * * *` （UTC 0:00 = JST 9:00）
   - **変更後**: `50 22 * * *` （UTC 22:50 = JST 7:50）
2. コメント等で時刻の説明が記載されている場合は、併せて「毎日午前7時50分（日本時間）」など、実態に合うよう修正してください。

## 【厳守事項】
- 本件は **`update_wiki.yml` のみの修正タスク** です。
- すでに実装され正常に動作しているPythonスクリプト（`src/` ディレクトリ配下）や、その他の設定ファイルには**一切の変更を加えないでください**。
